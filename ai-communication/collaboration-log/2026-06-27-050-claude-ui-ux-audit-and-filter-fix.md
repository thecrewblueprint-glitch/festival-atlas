## Issues found and fixed

### 1. Nav href prefix breaking validation (all 12 required pages)

ChatGPT's recent commits changed all nav links to use `href="./index.html"` prefix format. The static app validator checks for `href="index.html"` (no `./` prefix) and requires a separate Guide nav link on all required pages. ChatGPT had also removed Guide from the simplified nav.

Fixed: Python script replaced `href="./` → `href="` on all 12 required pages and inserted `<a href="guide.html">Guide</a>` after the Home link on each. Guide.html received `class="active"` on its own Guide link.

### 2. Collaboration log Commit metadata missing (logs 044, 045, 046, 048, 049)

Static app validator requires `/^Commit: /m` on all collaboration log files. Five recent ChatGPT logs were missing this line.

Fixed: Added `Commit: <hash>` line after `Branch: research-version` in each log.

### 3. State dropdown missing from Opportunities filter bar (user-reported)

Root cause: `atlas.css` contained a global rule:

```css
.filters #branchFilter,.filters #regionFilter,.filters #stateFilter,.filters #employerTypeFilter{display:none!important}
```

This was added to scope the public filter bar to search and date only, but it suppressed `#stateFilter` on opportunities.html where it is intentional. The `forceVisible()` inline style workaround in `opportunities-promoter-filter.js` could not override `!important` CSS.

Fixed:
- Removed `#stateFilter` from the CSS hide rule. Branch/region/employerType filters remain hidden.
- Updated filter bar grid to `1fr 1fr 1fr auto` to fit producer, state, month, reset in one row.
- Removed `installFallbackStatePanel()` and `fillFallbackStatePanel()` workaround functions and all calls to them from `opportunities-promoter-filter.js`.
- Bumped cache version to `?v=promoter4`.

## Remaining considerations

- The US SVG basemap in `map-page-static.js` is a simplified polygon. A more detailed SVG could improve recognizability but is not a blocker.
- On the calendar, `min-width:920px` requires horizontal scroll on narrow mobile. The `overflow:auto` wrapper handles this. Acceptable for a calendar grid view.
- The `opportunities-promoter-filter.js` and `atlas-core-v2.js` both populate and listen to `#stateFilter` on the opportunities page. Core fills basic state options; promoter filter replaces them with multi-market-aware options. Core also attaches an `input` listener to `#producerFilter` (since it selects all `#filters select` elements), triggering `renderPage()`. The MutationObserver then re-applies producer filtering. No user-visible issue, but a clean integration into core would remove this layering.
